## 개요

이 문서는 현재 사주 오행/천연석 추천 서비스에 새로 추가할 기능 페이지와 개발 범위를 정리한 기획 문서입니다. 다음 개발 단계에서는 계산 정확도, 해석 깊이, 모바일 사용성을 함께 개선합니다.

## 1. 절기 시각 기반 월주/년주 정밀 보정 완료 - 다음 개발 제외

### 개발 대상 페이지

- 메인 입력 페이지: `saju-five-elements/index.html`
- 상세 만세력 리포트 영역
- 계산 엔진: `saju-five-elements/app.js`

### 기능 목적

현재 월주는 고정 절기 날짜를 기준으로 산출합니다. 다음 단계에서는 절기 날짜뿐 아니라 절기 시각까지 반영하여, 절기 당일 출생자의 월주와 년주가 더 정확하게 계산되도록 보정합니다.

### 핵심 요구사항

- 출생 연월일시를 기준으로 절기 경계 전후를 판별합니다.
- 입춘 이전 출생자는 사주 년주가 전년도 기준이 될 수 있도록 보정합니다.
- 월주는 각 월의 절입 시각 이후부터 다음 월주로 전환합니다.
- 절기 시각 데이터가 없는 경우 현재 고정 절기 날짜 기준으로 fallback 처리합니다.
- 리포트에 `절기 보정 적용 여부`, `기준 절기`, `절기 시각`을 표시합니다.

### 개발 방향

- 절기 데이터 구조를 `monthBoundaries`에서 `solarTermBoundaries` 형태로 확장합니다.
- 데이터 예시는 `year`, `term`, `dateTime`, `monthBranch`, `note` 필드를 사용합니다.
- KASI API 또는 내부 절기 테이블을 연결할 수 있도록 계산 함수를 분리합니다.

### 디자인 방향

- 상세 리포트 상단에 작은 상태 배지로 `절기 보정 적용`을 표시합니다.
- 절기 경계 근처 출생자는 안내 문구를 보여줍니다.
- 계산 근거는 긴 문장보다 `기준 절기 / 절기 시각 / 적용 결과` 3열 요약으로 제공합니다.

### 완료 기준

- 입춘 전후 출생 테스트에서 년주가 올바르게 달라집니다.
- 절입 시각 전후 출생 테스트에서 월주가 올바르게 달라집니다.
- 절기 데이터가 없어도 기존 계산이 중단되지 않습니다.

### 완료 처리

- 상태: 완료
- 다음 개발 제외: 예
- 반영 파일: `saju-five-elements/app.js`, `saju-five-elements/styles.css`, `plan.md`
- 구현 내용: 태양 황경 기준 절기 시각 계산, 입춘 기준 년주 보정, 절입 기준 월주 보정, 상세 리포트 절기 보정 카드 추가

## 2. 지장간/십성/신살 계산 로직 고도화

### 개발 대상 페이지

- 상세 만세력 리포트 영역
- 사주 상세 해설 영역
- 계산 엔진: `saju-five-elements/app.js`

### 기능 목적

현재 리포트는 오행과 기본 간지 중심입니다. 다음 단계에서는 지지 속 지장간, 일간 기준 십성, 주요 신살을 함께 계산하여 사용자가 자신의 사주 구조를 더 깊게 이해할 수 있도록 합니다.

### 핵심 요구사항

- 지지별 지장간을 표시합니다.
- 일간을 기준으로 각 천간과 지장간의 십성을 계산합니다.
- 주요 신살을 우선순위별로 표시합니다.
- 신살은 과도하게 불안감을 주는 표현을 피하고, 해석형 문장으로 순화합니다.
- 신살 계산 근거와 적용 위치를 함께 보여줍니다.

### 우선 적용 항목

- 지장간: 자, 축, 인, 묘, 진, 사, 오, 미, 신, 유, 술, 해 전체 매핑
- 십성: 비견, 겁재, 식신, 상관, 편재, 정재, 편관, 정관, 편인, 정인
- 신살 1차: 도화, 역마, 화개, 장성, 천을귀인
- 신살 2차 후보: 문창, 양인, 괴강, 공망

### 개발 방향

- `hiddenStemsByBranch`, `tenGodByStemRelation`, `symbolicStars` 같은 독립 데이터 맵을 만듭니다.
- 계산 함수는 화면 렌더링과 분리해 테스트하기 쉽게 구성합니다.
- 상세 해설은 `강한 구조`, `보완할 구조`, `활용 포인트`로 나눕니다.

### 디자인 방향

- 원국 표에는 지장간을 작은 칩 형태로 표시합니다.
- 십성은 천간/지지 아래에 보조 텍스트로 보여줍니다.
- 신살은 카드형 목록으로 만들고, 긍정적 활용 문장을 먼저 보여줍니다.

### 완료 기준

- 일간이 바뀌면 십성 결과가 함께 바뀝니다.
- 지지별 지장간이 누락 없이 표시됩니다.
- 신살 결과에는 적용된 기준 지지 또는 천간이 함께 표시됩니다.

## 3. 모바일 상세 리포트 표 UX 추가 개선

### 개발 대상 페이지

- 상세 만세력 리포트 영역
- 대운/세운/원국 표
- 모바일 반응형 스타일: `saju-five-elements/styles.css`

### 기능 목적

상세 만세력 리포트는 정보량이 많아 모바일에서 표를 읽기 어렵습니다. 다음 단계에서는 표를 단순 축소하지 않고, 모바일에서 비교와 탐색이 쉬운 구조로 개선합니다.

### 핵심 요구사항

- 모바일에서 표 첫 열 또는 핵심 열을 sticky 처리합니다.
- 대운/세운 표는 가로 스크롤 위치를 인지할 수 있게 합니다.
- 작은 화면에서는 표와 카드형 요약을 함께 제공합니다.
- 현재 대운과 현재 연도는 시각적으로 강조합니다.
- 셀 안의 긴 텍스트가 줄바꿈되어도 레이아웃이 깨지지 않아야 합니다.

### 개발 방향

- `.report-table-wrap`에 스크롤 힌트와 shadow 상태를 추가합니다.
- 모바일에서는 `min-width`를 유지하되 행 높이와 폰트 크기를 조정합니다.
- 중요한 정보는 표 위 카드 요약으로 한 번 더 제공합니다.
- 터치 사용성을 위해 셀 padding과 행 간격을 재조정합니다.

### 디자인 방향

- 표는 과한 장식 없이 밝은 배경과 얇은 구분선 중심으로 정리합니다.
- 현재 대운/현재 세운은 부드러운 강조색 배지로 표시합니다.
- 모바일에서는 한 화면에 `현재 위치`, `핵심 간지`, `해석 키워드`가 먼저 보이게 합니다.

### 완료 기준

- 375px 모바일 화면에서 표 텍스트가 겹치지 않습니다.
- 대운/세운 표의 현재 행을 즉시 인지할 수 있습니다.
- 가로 스크롤이 필요한 영역과 필요 없는 영역이 명확히 구분됩니다.
